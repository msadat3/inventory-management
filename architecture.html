Factory Inventory Management System

Architecture overview — a Vue 3 + FastAPI demo with in-memory mock data.

Frontend: localhost:3000 Backend: localhost:8001 Data: JSON files in server/data/

System Architecture

Three-tier layout: a Vue 3 single-page client, a FastAPI REST service, and a file-backed in-memory data layer. No database; JSON is read once into Python lists at startup.

Data Flow — Request Lifecycle

What happens when a user changes a filter on the Dashboard.

  1. User interaction in a Vue view

    A filter (warehouse, category, status, or time period) is updated in App.vue. A composable exposes the current filter refs to child views.

  2. View triggers a reload

    A watch on filter refs calls a loader (e.g. loadData()) inside the view's setup(). Loading state flips to true.

  3. api.js builds the request

    client/src/api.js assembles query params with URLSearchParams, skipping any filter set to 'all', and calls axios.get(`${API_BASE_URL}/...`).

  4. FastAPI route handler

    main.py receives the request, binds query params via typed Optional[str] args, and hands off to apply_filters() and filter_by_month().

  5. In-memory filtering

    Python list comprehensions walk the lists loaded from JSON at startup. Quarters (Q1-2025 etc.) are expanded to their three months via QUARTER_MAP.

  6. Pydantic validates the response

    The endpoint returns a list; FastAPI coerces each item through its response_model (e.g. List[Order]) before serializing to JSON.

  7. Vue stores raw data, derives the rest

    Results land in refs (allOrders, inventoryItems, …). Derived values — totals, groupings, chart series — are computed, so they recache only when their inputs change.

  8. Render

    The template re-renders from the updated refs and computed properties. Loading flips back to false; errors, if any, surface via v-else-if.
